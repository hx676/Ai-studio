## 16. UX 反馈分层与共享弹窗

- [ ] 在 API 设置页触发缺少必填项的保存或测试连接，错误显示在对应字段附近，字段获得焦点，页面不只弹出笼统提示。
- [ ] 在任一成功保存场景触发 toast，toast 文案短、可读，自动消失，不遮挡当前表单输入。
- [ ] 在生成类页面触发长任务，页面显示持续状态或日志，提交按钮进入 loading 或 disabled 状态，再次点击不会重复提交。
- [ ] 在生成失败或连接失败时，页面显示用户能理解的摘要；如有技术详情，能展开或复制，不直接把原始堆栈当作唯一提示。
- [ ] `StudioDialog.alert('检查')` 可从浏览器控制台打开弹窗，OK 按钮默认可聚焦，关闭后焦点回到打开前的控件。
- [ ] `StudioDialog.confirm('删除？')` 打开后按 Tab，焦点停留在弹窗内；按 Escape 关闭后，页面没有残留遮罩。
- [ ] `StudioDialog.toast('已保存', { severity: 'success' })` 或等效调用出现成功样式；错误 toast 与成功 toast 视觉状态可区分。
- [ ] toast 容器有 live region 语义，使用浏览器无障碍树或 DOM 检查能看到 `aria-live` 或同等属性。

## 17. 画布与智能画布高风险交互

- [ ] 普通画布中选中 1 个节点按 Delete 后，节点删除，并出现“已删除 1 个节点，可撤销”或等效 toast。
- [ ] 普通画布删除后点击 toast 中的撤销操作或按 Ctrl+Z，节点恢复到删除前位置。
- [ ] 普通画布中焦点在输入框、文本节点或 contenteditable 区域时按 Delete 或 Backspace，只编辑文本，不删除画布节点。
- [ ] 普通画布多选删除时，toast 或确认文案包含删除数量，不只显示“已删除”。
- [ ] 普通画布永久删除、回收站清空或删除生成结果等高风险操作，必须先出现确认弹窗，取消后数据仍在。
- [ ] 智能画布中新建节点后，composer 出现在点击位置附近或节点附近，不跑到视口外。
- [ ] 智能画布创建图片节点、提示词节点、循环节点时，控制台无 `ReferenceError`，页面能继续编辑。
- [ ] 智能画布节点创建、复制、删除、保存资产等操作有对应通知，成功和失败状态能区分。
- [ ] 智能画布删除 1 个节点后可撤销；删除多个节点时文案包含数量；高风险删除出现确认或更强警告。
- [ ] 智能画布拖动节点进行连接、合并或保存资产时，放下前有操作标签或提示，放下后有完成或失败反馈。
- [ ] 画布和智能画布的快捷键帮助可发现，打开后 Escape 能关闭，关闭后焦点或键盘操作恢复正常。
